# 初期ファームウェアと違うところ
## ディレクトリ構成が変わりました。
ZMK Modulesのお作法に従ってファイルの置き場所を変更しています。  
初期ファームウェアでも今のところ問題はありませんでしたが、  
先のことを見据えて対応しています。
## トラックボールのドライバ関連
zmkデフォルトのPMW3610のドライバは超高速で通信するため、  
MCUにものすごい負荷がかかり、ファームウェア全体が不安定になる場面がありました。  
そのため、こちらはbadjeff氏のzmk-pmw3610-driverを導入しています。  
導入に伴い、関連するパラメータ等が全部変わっています。  

おまけに、起動時の不安定さがなくなったので  
Cmakeで読み込んでいたdelay_hack(ファームウェアの起動を遅らせる機能)も不要になりました。  
しばらく使って検証していますが、100%を保証することはできないので  
不安定だなと思ったらbuild.ymlのcmakeの頭についてる#を消してください。
delay_hackが復活します。
## R側のoverlayにいろいろ書き加えています
デフォルトでは一応オフにしていますが、  
スクロールに割り当てたり、ポインタ移動速度を変えるためのinput-processorsを書いてあります。  
書き換え方のメモもつけてあるので、いじりたい方の参考になれば幸いです。
## ボリューム上げ下げを割り当てました
keymapファイルにて、layer_2時に左のエンコーダを回すとボリュームが上げ下げできる記述を加えました。 
``` 
&inc_dec_kp C_VOL_DN C_VOL_UP  
```
何か書き換えたいと思った時の、書き方の参考を兼ねています。
## 余談
使用する構成が固まったときは、R側のoverlayにある  
split_listnerのうち、使用しない方をコメントアウト、あるいはステータスをdisabledに変更すると  
動作が更に軽くなると思います。
### 記入例1 全体をコメントアウト
```dts
/*
&glidepoint_split_listener {
    status = "okay";
};
*/
```
### 記入例2 1行ずつコメントアウト
```dts
//&glidepoint_split_listener {
//    status = "okay";
//};
```
### 記入例3 statusを変更
```dts
&glidepoint_split_listener {
    status = "disabled";
};
```
